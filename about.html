About MemoryJar

The idea, the technology, and the person behind it.

Why this exists

The big moments in life — graduations, weddings, trips abroad — already have photos and stories around them. The small ones don't. The good coffee, the late-night conversation, the small thing a friend said that made the day feel lighter — those slip away within a week.

MemoryJar is a small attempt to keep them. You drop a memory in the jar whenever life is sweet, and shake the jar later when you need a small lift. That's the whole product.

How it's built

MemoryJar is a static, multi-page web application written from scratch in plain HTML, CSS and JavaScript — no frameworks, no build step. That choice is deliberate: the site has to load instantly, work offline, and stay readable for someone reviewing the source code.

Memories are saved directly inside the browser using the localStorage Web API, which acts as a tiny per-device database. Nothing leaves the user's machine — there is no backend server, no account, and no tracking.

  • HTML5
  • CSS3 (Custom Properties, Grid, Flexbox)
  • JavaScript (ES Modules)
  • localStorage Web API
  • Google Fonts
  • crypto.randomUUID

The pages

Home — a quiet landing page with a glass-jar illustration, your live stats and your most recent four memories.

Add — a friendly form with a mood picker, category dropdown, paper-colour picker and date selector.

My Jar — the jar itself. Every memory is a little folded paper inside the glass; tap any paper to read it, or hit "Shake the jar" to pull a random one out.

Wall — the full collection in a grid, with live search, category chips, summary stats and a category bar chart.
